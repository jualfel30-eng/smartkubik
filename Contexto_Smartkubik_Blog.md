# Contexto Smartkubik Blog

## Producto
Smartkubik es un ERP SaaS multivertical, cloud-native, diseñado para PYMEs (10-100 empleados).
Stack: NestJS + MongoDB + React + Next.js + WebSockets.
Mercado principal: Venezuela, expandiendo a LATAM.
Precio: USD, modelo SaaS mensual/anual.

## Audiencia
Dueños de PYMEs, gerentes de operaciones, emprendedores en crecimiento.
Verticales: Retail, Restaurantes, Servicios, Manufactura, E-commerce, Clínicas.
Perfil: Operan con Excel/WhatsApp/sistemas básicos. Necesitan orden sin complejidad.

## Tono y Estilo
- Voz: Venezolana, humana, cercana, con humor observacional ligero
- Narrativa: Storytelling 100%. Cada artículo es una historia con personajes reconocibles
- Estructura: Escenas, diálogos, sensaciones. El lector debe "estar ahí"
- Inspiración espiritual (sin nombrarlos): Kurzgesagt (claridad), Bryson (curiosidad), Veritasium (misconceptions)
- Vulnerabilidad estratégica: El narrador admite errores, comparte dudas, construye puente

## Pilares de Contenido
1. Compras/Inventarios/Costeo
2. Ventas/Órdenes/POS
3. Finanzas/Contabilidad
4. Operaciones/Logística/Servicios
5. CRM/Posventa/Automatización
6. Analítica/Reportes/BI
7. Nómina/RRHH/Productividad

## Objetivos de Contenido
- Educar primero, vender después (inbound puro)
- Generar confianza antes de conversión
- Optimizado para AI discovery (Claude, ChatGPT, Perplexity)
- Cada artículo debe tener descargable de alto valor

## Características Editoriales
- Longitud: ≥2,000 palabras por artículo
- Formato: Híbrido (narrativa profunda + estructura AI-friendly)
- CTAs: Suaves, no agresivos ("Prueba gratis", "Descarga kit", "Newsletter")
- Datos: Siempre con fuente cuando se mencionan estadísticas
- FAQs: Optimizadas para queries conversacionales

## NO queremos
- Hard sell o venta agresiva
- Claims absolutos sin evidencia
- Listas de bullets como contenido principal (solo condimento)
- Jerga técnica innecesaria
- Tono corporate o frío
